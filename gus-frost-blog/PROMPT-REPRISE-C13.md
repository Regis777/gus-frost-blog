**Décision pipeline : Drive → Cloudinary → Notion. Cloudinary EST requis** — le kit avait raison.

**Pourquoi (vérifié dans les scénarios en production) :** le scénario Publer (`6256355`) alimente Publer par un **CSV d'import** dont la colonne **« Media URL(s) »** est mappée sur le champ Notion **`Image URL`**. Publer a donc besoin d'une **URL publique** pour récupérer l'image ; il l'héberge ensuite dans sa médiathèque. C'est le scénario `6223624` (« Images → Cloudinary → Notion ») qui produit cette URL. **Sans Cloudinary, `Image URL` reste vide → le CSV Publer sort avec une Media URL vide → rien ne se publie.**

**Flux à 9 modules (Cloudinary ajouté le 16/07/2026) :**
1. `google-drive:watchFilesInAFolder` (limit 1)
2. `google-drive:getAFile` (`select:"value"` → binaire `{{2.data}}`)
3. `archive:UnpackAction` (champ = **`data`**, pas `sourceFile`)
4. `util:FunctionIncrement` (**`reset:"run"`** — le champ de sortie est **`i`**, pas `value`)
5. `util:SetVariable2` → `targetName` : `switch()` contenant **tous les noms de fichiers cibles**, sélectionnés par `get(split(...); ifempty(4.value; 4.i))`, la clé étant le nom du ZIP
6. `google-drive:uploadAFile` → `📤 PNG prêts`, nom `{{5.targetName}}`, **avec filtre `exist` sur `{{5.targetName}}`** (garde-fou : ignore les pages en trop ; il gate aussi tout l'aval)
7. `cloudinary:UploadResource` — connexion `8313330`, `file: {{3.data}}`, `type: upload`, `file_type: data`, `mime_type: image`, `resourceType: image`, `public_id: {{replace(5.targetName; ".png"; "")}}` (⚠️ **sans l'extension** : Cloudinary ajoute le format, sinon l'URL sort en `.png.png`) → renvoie `secure_url`
8. `notion:searchObjects1` (filtre `Nom fichier |&*^%$#@| rich_text` / **`rich_text:equals`** / `{{5.targetName}}`)
9. `notion:updateADatabaseItem` → `Image URL` = `{{7.secure_url}}` **et** `Statut` = **« À dater »**

### ⚠️ Cycle de vie des statuts (corrigé le 16/07/2026)
```
Brouillon → [Canva + pipeline 6324761] → À dater
          → [planificateur 6299952]    → À publier
          → [Publer 6261329]           → Programmé (Publer) → Publié
```
Le pipeline dit **« l'image est prête »** (→ À dater). C'est le **planificateur** qui dit **« et voici quand »** (→ À publier). Avant correction, le pipeline écrivait « À publier » directement et **court-circuitait la datation** — c'est la cause des dates périmées de C1.

### Le planificateur `6299952` (réécrit le 16/07/2026)
Il lit toutes les épingles **Cible = Pinterest + Statut = À dater**, triées par `Nom fichier`, puis via un `util:FunctionIncrement` (`reset:"run"`) calcule :
```
position = ((i-1) × 95) mod 379          ← round-robin, 379 est premier
jour     = floor(position / 8)           ← 8 épingles/jour
date     = 2026-08-01 + jour
Statut   → « À publier »
```
Comme les variantes d'une même URL sont **consécutives** au tri (`-v01`…`-v04`), le pas de 95 les éloigne de ~95 positions ≈ **11 jours** — la règle des 7 jours est tenue **par construction**, sans colonne `Decalage`.
**Vérifié :** permutation sans collision, écart min 11 jours, max 8/jour, 01/08 → 17/09/2026.

🔴 **RÈGLE ABSOLUE : le planificateur se lance UNE SEULE FOIS, quand les 379 épingles sont toutes en « À dater ».** Les constantes `95`, `379` et `8` dépendent du total. Le relancer sur un sous-ensemble, ou deux fois, produirait des dates en collision. Il faut donc **produire tous les clusters d'abord**, planifier ensuite.
⚠️ La colonne `Decalage` est **obsolète** (vide partout) — le planificateur ne l'utilise plus.

### Leçons de débogage (ne pas les réapprendre)
- Canva nomme les fichiers internes du ZIP « untitled » → **seul l'ordre des pages compte** (d'où le compteur). L'ordre est **fiable** : page 1 = ligne 1 du CSV (vérifié).
- `__IMTINDEX__` **n'est pas** accessible dans les formules.
- Le validateur `Make:validate_module_configuration` est précieux, **mais** son énumération d'opérateurs Notion est fausse : **`rich_text:equals` fonctionne en production** malgré un signalement d'indisponibilité.
- Technique de diagnostic très efficace : **injecter les variables dans le nom du fichier de sortie** (`idx_v={{4.value}}_i={{4.i}}.png`) pour voir leurs valeurs réelles.
- Vider `📥 ZIP à traiter` avant chaque dépôt (un « T1 (1).zip » casse la clé du switch).
- **Un seul ZIP par run** (le Watch en limit 1 n'en prend qu'un par exécution).
- Ne **pas** vider `📤 PNG prêts` entre les templates : les PNG s'y accumulent, c'est leur place.

## 9. Procédure Bulk Create (rodée — suivre à la lettre)

1. Importer les **16 fonds** dans Canva (menu **Imports**), une fois pour toutes.
2. Ouvrir le template `Tx`.
3. Bulk Create → **« Importer les données »** (⚠️ **jamais** « saisir manuellement » : l'insertion d'images y est cassée au-delà de la 1re ligne).
4. Charger le CSV → les colonnes texte se remplissent seules.
5. **Ajouter une colonne Image** (le tableau doit déjà avoir ses lignes), puis insérer le bon fond **ligne par ligne** en suivant la colonne `Fond`.
6. **CONNECTER les données** — ⚠️ **l'erreur la plus fréquente, celle qui fait perdre du temps** : clic droit sur chaque élément → « Connecter les données » →
   - accroche → `Accroche`
   - CTA → `CTA visuel`
   - chiffre (T3) → `Chiffre`
   - **cadre de fond → `Image`**
7. Générer → **vérifier que le sujet n'est pas couvert par le texte**.
8. **Supprimer les pages vides** (Canva exporte TOUTES les pages, y compris vides — c'était la cause du bug « 22 pages au lieu de 8 »).
9. Exporter en PNG → nommer le ZIP exactement **`C13_T1.zip`** … **`C13_T8.zip`**.
10. Déposer dans `📥 ZIP à traiter` → lancer le scénario `6324761`.

**Ordre de production recommandé :** templates simples d'abord — **T2 → T4 → T5 → T6 → T7 → T1 → T3 → T8**.

**Technique Canva utile :** l'effet **« Arrière-plan »** sur un champ texte fait tracer par Canva le panneau coloré **autour du texte réel**, quelle que soit la longueur de l'accroche. Déjà appliqué sur les 8 templates. Réglages : arrondi 10-20 %, marge interne généreuse, opacité 100 %. Surveiller l'effet « marches d'escalier » sur le multiligne.

**Piège API Canva :** un **cadre vide est invisible** (`fills:[]`) — si une inspection ne montre « aucun cadre », c'est normal tant qu'il est vide.
